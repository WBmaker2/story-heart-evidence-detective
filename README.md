# 이야기 마음 근거 수사대

이야기 속 행동·표정·대사를 단서로 인물의 마음을 짐작하고, 한 장면에 여러 해석이 가능함을 살펴보는 한국어 읽기 웹앱입니다.

## 활동과 해석 모델

연습 사건 뒤에 여섯 개의 고정 사건을 차례로 읽습니다. 마음 하나와 서로 다른 단서 카드 두 장을 연결하면 수사 문장이 만들어지고, 제출 뒤에는 학생 친화적 안내와 다른 해석 하나가 나타납니다. 마지막 화면은 사건 완료와 사용한 단서 종류만 보여 줍니다.

해석 결과는 AI나 무작위 채점이 아닙니다. 정적 콘텐츠에 선언된 `reviewedReadings`와 사람이 미리 구성한 근거 카드 조합을 순수 함수가 같은 순서로 비교하므로 같은 입력은 항상 같은 결과를 냅니다. 검수 지도에 없는 선택도 절대적으로 틀렸다고 단정하지 않고, 현재 제공된 이야기와 근거 카드 안에서 연결 정도만 안내합니다.

## 교육과정 연결과 한계

- `[2국02-04]`의 인물 마음·생각 짐작 활동과 부분적으로 연결됩니다. 자기 경험과 비교하는 활동은 개인정보와 저학년 부담을 줄이기 위해 앱에서 다루지 않으므로 성취기준 전체 달성을 주장하지 않습니다.
- `[2국05-03]`의 인물 모습·행동·마음 상상과 부분적으로 연결됩니다. 시·노래·이야기·그림으로 창작하여 표현하는 활동은 제공하지 않습니다.
- 근거 카드 두 장을 고르는 규칙은 앱의 읽기 발판이며 교육과정의 별도 채점 기준이 아닙니다.

## 개인정보와 네트워크 모델

앱은 이름, 학급, 자유 입력, 개인 감정, 로그인 정보, 사진, 음성을 받지 않습니다. 진행 상태는 React 메모리에만 있으며 새로고침하면 초기화됩니다. `localStorage`, `sessionStorage`, 쿠키, 분석, 외부 API, 카메라, 마이크를 사용하지 않고 초기 앱 자원 외 외부 런타임 요청을 만들지 않습니다.

## 실행과 자동 검증

Node.js 22.13 이상이 필요합니다.

```bash
npm install
npm run dev
```

전체 실용 자동 게이트와 새 프로덕션 빌드는 다음과 같이 실행합니다.

```bash
npm run verify
npm run build
```

`verify`는 타입 검사, ESLint, 500줄 미만 파일 제한, 콘텐츠·본문 앵커 무결성, 개인정보·외부 요청·학생용 표현·스타터 잔여물 정적 검사, 단위 테스트, 빌드, 서버 렌더링 HTML 검사를 순서대로 수행합니다. 개별 명령은 `npm run typecheck`, `npm run lint`, `npm run test`, `npm run check:content`, `npm run check:file-lengths`, `npm run check:release`입니다.
