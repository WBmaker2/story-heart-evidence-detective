# 초등학생 실사용 QA 개선 구현 계획

- 작성일: 2026-07-16
- 대상 버전: 1.1.8
- 근거: `.gstack/qa-reports/qa-report-localhost-2026-07-16.md`
- 승인 기록: 사용자가 “찾은 문제를 개선해줘. 구현 시작”이라고 요청하여 계획 검토 대기 없이 구현을 진행한다.

## 목표

초등학교 2학년 사용자가 이야기 본문을 놓치지 않고, 읽기 실수를 쉽게 되돌리며, 쉬운 말로 추론 과정을 이해하고, 완료 화면에서 자신이 고른 마음과 단서를 돌아볼 수 있게 한다.

## 구현 범위

1. `다음 사건` 전환 시 새 이야기 제목으로 스크롤과 키보드 초점을 이동한다.
2. 문장별 읽기에 `이전 문장` 버튼을 추가하고 첫 문장과 마지막 문장에서 이동 범위를 지킨다.
3. 화면의 어려운 활동 용어를 저학년 중심 표현으로 바꾼다.
   - `대사` → `한 말`
   - `함께 검토` → `같이 살펴보기`
   - `다른 해석` → `다른 생각`
   - `수사 기록` → `내가 살핀 기록`
4. 완료 요약에 사건별로 고른 마음, 대표 단서, 확인한 다른 생각을 보여 준다.
5. 업데이트 내역에 1.1.8 개선 내용을 추가한다.

## 예상 변경 영역

- 사건·문장 이동 상태: `app/features/investigation/InvestigationApp.tsx`, `app/features/investigation/state.ts`
- 이야기 읽기 UI: `app/components/StoryPanel.tsx`
- 검토·요약 UI: `app/components/ReviewPanel.tsx`, `app/components/SummaryScreen.tsx`
- 요약 데이터: `app/domain/session.ts`, `app/domain/types.ts`
- 쉬운 말과 스타일: `app/features/investigation/feedback.ts`, `app/components/StartScreen.tsx`, 관련 CSS
- 변경 기록: `app/content/update-history.ts`
- 회귀 테스트: `tests/domain`, `tests/ui`, `tests/content`, 렌더링 테스트

## 완료 기준

- 402px 모바일에서 사건 1→2와 2→3 전환 직후 새 이야기 제목과 첫 문장이 화면 안에 보인다.
- 문장별 읽기 2/5에서 이전·다음 버튼을 모두 사용할 수 있고, 1/5와 5/5에서는 해당 방향만 비활성화된다.
- 주요 학습 화면에 `대사`, `다른 해석`, `함께 검토`, `수사 기록`이 사용자용 핵심 문구로 남지 않는다.
- 완료 요약의 각 사건에 고른 마음과 대표 단서 문장이 표시되고 점수·정답·오답 표현은 생기지 않는다.
- 초기화, 단서 2개 제한, 다시 살펴보기의 선택 유지가 회귀하지 않는다.
- `npm run verify`와 WebKit 데스크톱·모바일 실사용 흐름이 통과한다.

## 제약과 위험

- 기존 작업 트리의 1.1.7 미커밋 변경을 보존하고 그 위에 최소 변경으로 구현한다.
- 점수 없는 복수 해석 학습 원칙과 기존 사건 데이터 구조를 유지한다.
- 초점 이동은 마우스·터치 사용자를 방해하지 않으면서 화면 낭독기에도 새 사건 시작을 알려야 한다.
- 요약 정보가 길어져도 320~402px 화면에서 가로 넘침이 생기지 않아야 한다.
- 모든 소스 파일은 500줄 미만을 유지한다.

## 검증 계획

1. 상태·요약·문구 단위 테스트를 먼저 추가하거나 갱신한다.
2. 타입 검사, 린트, 콘텐츠·릴리스·파일 길이 검사와 전체 테스트를 실행한다.
3. WebKit에서 데스크톱 전체 흐름과 402×681 모바일 사건 전환을 직접 클릭해 확인한다.
4. 콘솔 오류, 실패 요청, 가로 넘침, 초점 위치를 확인하고 전후 스크린샷을 남긴다.

## 작업 기록

- 구현 전: QA 보고서의 High 1건, Medium 3건을 범위로 확정했다.
- 구현 중:
  - 새 사건 진입 시 이야기 제목에 초점을 주고 `scrollIntoView`로 이야기 시작 위치를 보여 주도록 했다.
  - 문장별 읽기에 이전 문장 동작과 양쪽 경계 처리를 추가했다.
  - 사용자 화면의 `대사·다른 해석·함께 검토·수사 기록`을 저학년용 표현으로 바꿨다.
  - 완료 요약 데이터에 고른 마음, 대표 단서, 실제 검토에서 이어진 다른 생각을 추가했다.
  - 업데이트 내역에 1.1.8을 추가했다.
- 검증:
  - `npm run verify` 통과: 단위 테스트 49개, 서버 렌더링 테스트 2개, 타입·린트·콘텐츠·릴리스·파일 길이·빌드 통과.
  - WebKit 기반 인앱 브라우저에서 402×681 모바일과 1280×900 데스크톱을 직접 확인했다.
  - 문장별 읽기 1/6→2/6→1/6, 사건 1~6 전체 완주, 모바일 사건 전환 5회, 완료 요약 6개를 확인했다.
  - 모든 모바일 사건 전환에서 새 이야기 제목 상단 18px, 제목 초점, 가로 넘침 없음이 확인됐다.
  - 콘솔 오류·경고와 프레임워크 오류 화면은 0건이었다.
- 계획 대비 검토:
  - 계획한 네 개선 항목과 1.1.8 기록을 모두 구현했으며 범위 변경은 없다.
  - 점수 없는 복수 생각 원칙, 단서 두 개 제한, 수정 시 선택 유지, 초기화 계약을 보존했다.
  - 실제 초등학생 관찰 및 실제 iPad VoiceOver·Android TalkBack 검증은 별도 수동 확인으로 남는다.
