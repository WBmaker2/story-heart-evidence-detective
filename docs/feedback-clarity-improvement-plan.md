# 피드백 판정·문구 명확성 개선 계획

## 배경

결과 화면에서 이미 선택한 단서인 `“괜찮아, 다시 갈게!”`를 다시 “함께 살펴볼 수 있다”고 안내해, 학생이 새 단서를 추천받은 것처럼 오해할 수 있다. 또한 선택한 두 단서가 각각 같은 마음과 관련되어도 서로 검토된 한 쌍이 아니면 첫 번째 일치 단서만 보고 “단서 한 장”이라고 판정한다.

## 목표

- 학생이 고른 단서와 새로 제안하는 단서를 명확히 구분한다.
- 두 단서가 각각 마음과 이어지는 경우와 단서 하나만 이어지는 경우를 구분한다.
- “정답/오답” 대신 현재 연결에서 확인된 점과 더 살펴볼 점을 구체적으로 설명한다.
- 모든 사건과 가능한 마음·단서 조합에서 같은 오해가 반복되지 않는지 자동 점검한다.

## 구현 범위

1. 전체 사건의 모든 마음 후보와 단서 두 장 조합을 판정해 피드백 유형별 사례 수를 확인한다.
2. 부분 일치 결과에 일치한 단서 ID를 모두 담도록 판정 모델을 개선한다.
3. 단서의 저장 순서에 따라 생성하던 일반 문구를 없애고, 판정 상황과 선택 여부가 드러나는 문구로 바꾼다.
4. 결과 화면에서 다음을 분명하게 안내한다.
   - 두 단서가 함께 마음을 뒷받침하는 경우
   - 두 단서가 각각 마음과 관련되지만 검토된 한 쌍은 아닌 경우
   - 선택한 단서 중 하나만 마음과 관련되는 경우
   - 다른 마음과 더 가까운 단서 조합인 경우
   - 아직 뚜렷한 연결을 찾기 어려운 경우
5. 문제를 발견한 `두 번째 출발선` 조합과 전체 데이터 전수 검사를 회귀 테스트로 추가한다.
6. 업데이트 내역에 피드백 설명 개선을 기록한다.

## 검증 기준

- 이미 선택한 단서를 “새로 함께 살펴볼 단서”처럼 표현하지 않는다.
- `“괜찮아, 다시 갈게!”`와 `“친구들을 바라보며 입술을 꼭 다물었습니다”`를 선택하면 두 단서가 각각 마음과 관련됨을 안내한다.
- 기존 네 가지 판정의 우선순위와 순서 독립성이 유지된다.
- 타입 검사, 린트, 콘텐츠 검사, 파일 길이 검사, 전체 테스트, GitHub Pages 정적 빌드가 모두 통과한다.
- 데스크톱과 모바일 결과 화면에서 문구가 읽기 쉽고 잘리지 않는다.
