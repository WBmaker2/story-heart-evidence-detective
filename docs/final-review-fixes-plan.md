# 최종 전체 브랜치 리뷰 수정 계획

작성일: 2026-07-15  
대상 브랜치: `codex/story-heart-mvp`  
기준 커밋: `2f0cec2`

## 목표

제품 범위를 늘리지 않고 최종 리뷰에서 확인된 세 가지 정확성 문제를 수정한다.

1. 조사 문장의 주격 조사 오류를 없애고 도메인 결과와 화면 미리보기의 문구를 일치시킨다.
2. 문장별 읽기 모드를 실제 한국어 문장 단위로 나누되, 인용된 물음표·느낌표와 뒤따르는 서술을 한 문장으로 보존한다.
3. 워커 릴리스 감사에서 같은 파일의 안전한 호출 하나가 안전하지 않은 다른 `ASSETS.fetch` 호출을 가리지 못하게 한다.

## 구현 원칙

- 조사 문장은 이름 뒤 조사를 동적으로 붙이지 않는 자연스러운 표현을 사용한다.
- 문장 분절은 부작용 없는 순수 헬퍼로 구현하며 원문 문자와 순서를 보존한다.
- 화면의 문장 수와 다음 문장 이동 한계는 모두 동일한 분절 결과를 사용한다.
- 워커 자산 요청은 전용 헬퍼로 모으고 감사기는 모든 `ASSETS.fetch` 호출을 개별 검증한다.
- 제품 변경은 업데이트 내역에 새 버전으로 기록한다.
- 모든 코드 파일은 500줄 미만을 유지한다.

## TDD 및 검증

1. 모음 끝 이름 `지호`와 자음 끝 이름 `민준`이 모두 자연스러운 조사 중립 문장으로 생성되는 실패 테스트를 먼저 추가한다.
2. 한 문단의 두 서술문, 인용 물음표·느낌표 뒤 서술 보존, 분절 재조합 원문 보존 테스트를 먼저 추가한다.
3. 안전한 호출과 안전하지 않은 호출이 함께 있는 워커 소스가 감사를 실패하는 회귀 테스트를 먼저 추가한다.
4. 각 실패를 확인한 뒤 최소 구현으로 GREEN을 만든다.
5. 전체 `npm run verify`와 별도 `npm run build`를 실행한다.
6. 변경 diff를 실행 순서대로 자체 리뷰하고 커밋한 뒤 `/private/tmp/story-heart-final-fixes-report.md`에 RED/GREEN, 파일, 결과, 커밋, 우려 사항을 기록한다.
