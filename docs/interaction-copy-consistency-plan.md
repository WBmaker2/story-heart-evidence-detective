# 이야기 마음 근거 수사대 단계·안내 문구 일관성 개선 계획

## 목표

초등 저학년 학생이 각 화면에서 지금 해야 할 행동을 한 번에 이해하도록, 실제 선택 규칙과 화면의 제목·단계 수·번호 표현을 일치시킵니다.

## 확인된 문제

1. 마음 하나만 고르는 화면의 큰 제목이 `두 단서로 짐작해 보세요`라고 먼저 안내해, 현재 화면에서 두 항목을 골라야 하는 것처럼 보입니다.
2. 시작 화면은 `수사 방법 세 단계`로 안내하지만 실제 진행 막대는 `이야기 읽기 → 마음 고르기 → 단서 연결 → 함께 검토`의 네 단계입니다.
3. 상단 진행 막대의 단계 번호와 본문 카드 안의 `1 마음 고르기`, `2 단서 2개 고르기`, `3 수사 문장` 번호가 서로 다른 체계를 사용해 현재 위치를 혼동하게 합니다.

## 구현 범위

- 현재 단계에 맞는 큰 질문을 보여 줍니다.
  - 이야기 읽기: 마음을 살필 장면을 읽도록 안내
  - 마음 고르기: 마음이나 생각 하나를 고르도록 안내
  - 단서 연결: 고른 마음과 이어지는 단서 두 개를 고르도록 안내
- 시작 화면의 수사 방법을 실제 진행과 같은 네 단계로 맞추고 `함께 검토`를 추가합니다.
- 본문 카드의 별도 순서 번호를 제거해 상단 진행 막대만 전체 단계 번호로 사용합니다.
- 점검 중 발견한 모바일 이야기 카드의 잘못된 고정 위치를 해제해 큰 빈 공간을 없앱니다.
- 기능 계약인 `마음 1개 + 서로 다른 단서 카드 2개`와 복수 해석 판정은 변경하지 않습니다.
- 업데이트 내역에 이번 개선 내용을 새 버전으로 기록합니다.

## 예상 변경 파일

- `app/components/InvestigationPanel.tsx`
- `app/components/StartScreen.tsx`
- `app/content/story-factory.ts` 또는 단계 질문을 소유할 별도 표시 함수
- `app/content/update-history.ts`
- 관련 UI·콘텐츠 테스트

## 수용 기준

- 마음 고르기 화면 어디에도 지금 단서 두 개를 선택하라는 상충 안내가 나타나지 않습니다.
- 단서 연결 화면은 마음이 이미 선택되었으며 단서 두 개를 골라야 한다고 분명히 안내합니다.
- 시작 화면과 상단 진행 막대가 모두 네 단계를 같은 순서와 이름으로 안내합니다.
- 한 화면에서 서로 다른 단계 번호 체계가 함께 보이지 않습니다.
- 마음은 라디오 버튼으로 하나만, 단서는 체크박스로 최대 두 개만 선택되는 기존 동작이 유지됩니다.
- 키보드 초점 이동과 단계 제목의 접근성 계약이 유지됩니다.
- 업데이트 내역에 2026-07-15 개선 기록이 추가됩니다.

## 검증 계획

1. 문구·단계 계약을 검증하는 UI 테스트를 추가하거나 보강합니다.
2. `npm run test:unit`, `npm run typecheck`, `npm run lint`를 실행합니다.
3. 전체 `npm run verify`로 빌드·콘텐츠·출시 점검까지 회귀 검증합니다.
4. 로컬 정적 화면에서 시작, 마음 고르기, 단서 연결, 함께 검토 흐름을 확인합니다.

## 위험과 제한

- 이야기별 판정 데이터와 학생 기록 구조는 건드리지 않습니다.
- 단계 제목 변경으로 초점 대상이 사라지지 않도록 기존 `data-stage-heading`을 유지합니다.
- 본문 안의 번호 제거가 시각적 여백을 깨뜨릴 수 있으므로 데스크톱과 작은 화면 레이아웃을 함께 확인합니다.

## 작업 기록

- 2026-07-15: 사용자가 전체 점검과 구현 시작을 요청해 계획 검토 대기 없이 구현 진행을 승인한 것으로 기록합니다.
- 2026-07-15: WebKit 모바일 검증에서 이야기 카드의 `position: static` 규칙이 CSS 불러오기 순서 때문에 무효화되는 문제를 발견해 모바일 레이아웃 수정을 범위에 추가했습니다.

## 구현 기록

- 단계별 큰 질문을 분리하고 마음 단계에서 `하나`, 단서 단계에서 `두 장`을 명시했습니다.
- 시작 안내와 진행 막대를 같은 네 단계로 맞추고 본문 안의 중복 번호를 제거했습니다.
- 모바일 단일 열 구간에서 이야기 카드를 고정하지 않도록 CSS 규칙 위치를 바로잡았습니다.
- 업데이트 내역 `1.1.7`과 문구·단계·모바일 회귀 테스트를 추가했습니다.

## 검증 기록

- 관련 UI·콘텐츠·상태 테스트 16개 통과
- 최종 `npm run verify` 통과: 단위 테스트 47개, 서버 렌더링 테스트 2개, 타입 검사, 린트, 500줄 제한, 콘텐츠·출시 점검, 빌드
- WebKit 데스크톱에서 시작 → 마음 고르기 → 단서 연결 흐름과 단계별 질문 확인
- WebKit 모바일 402px에서 가로 넘침 없음, 이야기 카드와 선택 카드 사이 간격 22px 확인

## 계획 대비 구현 검토

- 기능 계약과 판정 데이터는 변경하지 않았고 계획한 문구·단계·번호 개선을 모두 반영했습니다.
- 계획 작성 뒤 추가된 변경은 모바일 검증에서 발견한 기존 빈 공간 문제 한 건이며, 전체 화면 점검 요청과 계획의 모바일 위험 검증 범위 안에서 수정했습니다.
- 미완료 항목은 커밋·푸시·배포이며 이번 구현 요청 범위에서는 로컬 변경으로 남겼습니다.
